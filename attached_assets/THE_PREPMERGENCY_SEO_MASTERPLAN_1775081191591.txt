3.6 Core Web Vitals — what to target
Your app is Vite + React, which is already fast. Key checks:

LCP < 2.5s — ensure hero image/video on home loads fast; use loading="eager" on above-fold images, loading="lazy" on everything else
INP < 200ms — avoid blocking JavaScript on first interaction
CLS = 0 — always set explicit width/height on images so layout doesn't shift when they load (your browse-hero.png, join-hero.png, missions-hero.png all need this)
Run PageSpeed Insights on prepmergency.com after deploying — aim for 90+ mobile score
3.7 Handle SPA crawlability
Since the app is tab-based (/?tab=browse), Google won't index those tabs independently. The solution: the blog compensates for this entirely. Each blog post is a separate crawlable URL that drives traffic in. The app itself doesn't need to be indexed tab-by-tab — it needs to be discovered via blog and landing pages.

PART 7 — INTERNATIONAL SEO
Since you're targeting all English-speaking teachers globally with no physical location:

Do not use hreflang tags — you have one English version, not separate UK/US/AU versions. Using hreflang incorrectly can hurt you.
Do target geographic keywords in blog content — write posts like "Emergency lesson plans for UK supply teachers" or "Sub plans for Australian relief teachers". These are specific, low-competition, and let Google understand your global relevance.
Spell things both ways in blog posts — "lesson plans" (universal), "supply teacher" (UK), "substitute teacher" (US), "relief teacher" (AU/NZ). One post can rank for all three geographies.
