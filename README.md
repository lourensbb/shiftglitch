# Synapse — Interactive Learning OS

> *A study-methods operating system that trains teenagers to learn — not just to revise.*

---

Synapse is not a quiz app. It is not a gamified to-do list. It is not a flashcard website with a leaderboard bolted on.

Synapse is a structured, evidence-based learning platform that teaches teenagers *how* to study — and then holds them accountable to doing it. Every tool inside it is grounded in cognitive science. Every interaction is designed to build a real study habit. The progression system exists not to reward hours spent clicking, but to recognise and certify that a student is genuinely using the techniques that actually work.

It is designed for teenagers who are serious about their results, parents who want to see structured progress, and teachers who need something more than "did they revise tonight?" as a feedback mechanism.

## 1. Product Philosophy

Most study apps make studying feel like a game. Synapse makes it feel like a discipline.

The distinction matters. Gamified apps reward *time on screen*. Synapse rewards *evidence of real learning behaviour*. A student does not advance in rank by earning XP for logging in. They advance by demonstrating — repeatedly, over time — that they are blurting concepts from memory, breaking large tasks into smaller ones when they get stuck, using spaced repetition to retain what they have studied, and completing knowledge diagnostics that reveal where the gaps actually are.

The app is built around five core convictions:

**1. Behaviour, not volume.** Forty hours of passive re-reading is worth less than four hours of active recall. Synapse tracks the right things.

**2. The technique must be taught, not assumed.** Every tool inside Synapse is a structured implementation of a known learning method — Pomodoro, Leitner, Feynman, Blurting, Eisenhower. Students are not just given a timer. They are given a framework.

**3. Progression must be earned, not purchased.** Rank advancement is permanent and irreversible. It cannot be bought, skipped, or hacked. The only path forward is demonstrated behaviour.

**4. Honesty about gaps.** The MCQ Diagnostic system surfaces knowledge gaps directly. The app does not hide weak areas or celebrate mediocre scores. It shows students exactly what they do not know yet, and gives them the tools to close those gaps.

**5. The teenager is the user.** Every design decision — dark mode by default, phone-lock nudges, lofi music integration, study buddy comparisons, growth mindset reframing on flashcards — exists because the target user is a distracted, phone-addicted, often anxious 14-to-18-year-old who needs structure, not judgment.

## 2. Who It Is For

### Students (Ages 13–19)
Synapse is designed for secondary school and early university students who are ready to take responsibility for their own learning. It works best for students who have exams to prepare for and who are willing to be honest about what they do and do not know.

### Parents
Parents can view a shareable progress report generated by the student — a URL that opens a read-only summary of their rank, evidence, and activity. No account needed on the parent's side. Clear, honest, and impossible to fake.

### Teachers & Classroom Educators
The Teacher Dashboard (`/teacher`) is a separate, standalone page. Students export their data and send it to the teacher as a JSON file. The teacher imports all files at once and instantly sees a class-wide overview: rank distribution, activity levels, students needing attention, and printable individual progress reports. No accounts, no subscription, no complexity.

## 3. The Rank Progression System

The rank system is the single authoritative spine of the Synapse experience. There are three ranks. Progression is permanent — once a rank is earned, it is never revoked. The requirement to advance is not a single task: all evidence criteria must be met simultaneously.

| Rank | Pomodoros | Blurts | Governor Breakdowns | Cards Advanced | Active Days | Diagnostics |
|------|-----------|--------|---------------------|---------------|-------------|-------------|
| **Space Cadet** | — | — | — | — | — | — |
| **Second Officer** | 15 | 10 | 10 | 50 | 7 | 2 |
| **Flight Commander** | 50 | 30 | 30 | 150 | 21 | 5 |

**What each evidence type means:**

- **Pomodoros Completed** — full 25-minute focus sessions completed using the Learning Governor
- **Blurts Completed** — completed Blurting Method sessions (active recall from scratch)
- **Governor Breakdowns** — times the student deliberately broke down a task when the Governor blocked them for repeating the same task 4 sessions in a row
- **Cards Advanced** — Leitner flashcards promoted to a higher box (a measure of genuine retention, not just card creation)
- **Active Days** — distinct calendar days with recorded study activity
- **Diagnostics Completed** — MCQ Diagnostic mission submissions (missions can be retried; each submission counts)

The rank badge is displayed in the navigation bar on every page. Rank evidence progress bars are visible on the Rank page, with a live breakdown of how far the student is from the next threshold.

## 4. Full Feature Reference

### Learning Governor (Pomodoro Timer)

The Learning Governor is the productivity engine at the heart of Synapse. It is based on the Pomodoro Technique but adds the Governor mechanism — a deliberate enforcement layer that prevents shallow repetition.

- **Fixed duration:** 25 minutes (focus) / 5 minutes (break)
- **Subject tagging:** Every session is tagged with a subject (Maths, English, Science, History, Geography, Languages, Art, Other)
- **Phone-lock nudge:** A notification fires the moment a session starts — a reminder to put the phone face-down
- **Lofi music link:** A single tap opens a YouTube lofi study playlist in a new tab
- **Circular countdown ring:** A visual progress ring counts down in real time
- **Governor blocking:** After 4 consecutive sessions on the same task, the Governor locks the timer and forces the student to either break the task down or switch to a different one. This is recorded as evidence.
- **"What I Learned?" modal:** At the end of every completed session, a modal asks the student to write one sentence about what they just learned. These entries are saved to a personal learning log.

---

### Flashcard Decks (Leitner Spaced Repetition)

Synapse's flashcard system implements the Leitner 5-box spaced repetition method — a system developed by the German journalist Sebastian Leitner in 1972 and still one of the most effective memory tools in existence.

**How it works:**
- Students create decks (any subject or topic)
- Within each deck, they create cards with a question (front) and answer (back)
- Each card lives in one of 5 boxes representing its stage of retention
- During a study session, cards are presented one at a time
- The student responds with one of three options:
  - **Again** — the card returns to Box 1 (most frequent review)
  - **Not Yet** — same as Again, but triggers the YET Growth Shield mindset reframe
  - **Got It** — the card advances to the next box

A visual orbit indicator shows each card's stability level. Box 5 cards are considered mastered. Cards in higher boxes are reviewed less frequently.

**Rank contribution:** Every time a card advances to a higher box, that counts as one `cardsAdvanced` evidence point.

---

### Blurting Method

The Blurting Method is a structured implementation of free recall — one of the most research-supported learning techniques available. Students attempt to write down everything they know about a topic *before* looking at their notes.

**The four phases:**

| Phase | What Happens |
|-------|-------------|
| **1. List** | Student writes every sub-topic or concept they can think of from memory, without any notes |
| **2. Read** | Student reviews their actual notes or textbook for the topic |
| **3. Blurt** | Student puts notes away and writes everything they can recall again |
| **4. Compare** | Student compares their blurt against the original notes to identify gaps |

Sessions are saved with the topic title, recall attempt text, and identified gaps. Review count tracks how many times a student has returned to the same topic.

**Rank contribution:** Each completed blurt session counts as one `blurtsCompleted` evidence point.

---

### Mission Architect (AI Study Planner)

The Mission Architect uses Google's Gemini AI model to generate personalised, structured study plans. The student inputs their subject, exam date, available study time, and any specific concerns. The AI returns a multi-week plan with specific daily tasks, technique recommendations, and topic priorities.

The AI key is **Bring Your Own Key (BYOK)** — each user enters their own free Google Gemini API key in Settings. The key is stored in their browser only and never sent to any external server. If no user key is set, the app falls back to a server-side proxy using the operator's key.

### MCQ Diagnostics

A 100-question multiple-choice knowledge assessment split across 5 thematic missions. This is not a general trivia quiz — every question is directly tied to the learning science and study methods that Synapse is built on. Students are expected to learn the theory, not just the tools.

Full mission breakdown in [Section 5](#5-the-mcq-diagnostic-missions).

---

### Babel Fish Metaphor Lab (Feynman Technique)

The Feynman Technique is the practice of explaining a concept as simply as possible — ideally to someone who knows nothing about it. If you cannot explain it simply, you do not understand it.

The Babel Fish Metaphor Lab is Synapse's implementation of this technique. Students choose a concept, write a plain-language explanation, optionally add a "stupid doodle" (a rough diagram or sketch), and save the session. The act of constructing the explanation forces the student to identify where their understanding breaks down.

Sessions are saved and reviewable. An optional doodle badge is awarded for including a visual element.

---

### Dopamine Recalibrator

Two anti-procrastination tools in one panel:

- **Stealth Mode** — removes all distractions from the student's workspace by hiding everything except the essential study surface. Designed for students who get distracted by their own screen clutter.
- **Entry Rocket** — a micro-commitment tool. The student writes one very small, very specific starting action (e.g. "Open my notebook to page 12"). The act of committing to the smallest possible step bypasses the psychological resistance of starting.

---

### Yet Growth Shield

A mindset intervention tool based on Carol Dweck's growth mindset research. When a student encounters something they do not understand, the Yet Growth Shield reframes the experience:

*"I don't understand photosynthesis"* → *"I don't understand photosynthesis **yet**."*

The tool is triggered in two ways: directly from the Yet Growth Shield page (for any concept the student wants to work through), or automatically when a student taps **Not Yet** on a flashcard. The concept is logged with a date. Students can review their full "yet" history.

---

### Eisenhower Matrix

An interactive task prioritisation tool based on the Eisenhower Decision Matrix — the productivity framework attributed to Dwight D. Eisenhower and popularised by Stephen Covey in *The 7 Habits of Highly Effective People*.

Students place tasks into one of four quadrants:

| | Urgent | Not Urgent |
|---|-------|-----------|
| **Important** | Do First | Schedule |
| **Not Important** | Delegate | Eliminate |

Tasks can be added, moved between quadrants, and deleted. A reflection prompt encourages the student to articulate why they placed a task in a particular quadrant.

---

### Cornell Notes

A structured digital note-taking panel based on the Cornell Note-Taking System developed at Cornell University. The panel is visible directly on the dashboard. Notes are auto-saved as the student types.

Optional cloud sync via Firebase allows notes to persist across devices. Without Firebase, notes are saved in `localStorage`.

---

### Quiz Arena

A standalone trivia game powered by the Open Trivia Database API. The Quiz Arena is entirely separate from the rank progression system — it is a low-stakes, informal tool for general knowledge. Performance in the Quiz Arena has no impact on rank evidence.

---

### Charts and Session Statistics

A dedicated Stats page provides visualisations and session history:

- **Focus time scatter plot** — each Pomodoro session plotted by date and duration
- **Weekly activity bar chart** — sessions per day over the last 7 days
- **Technique usage doughnut chart** — proportion of activity across different tools
- **Session history** — scrollable log of all focus sessions with date, subject, and duration
- **"What I Learned?" log** — all WIL entries from session-end prompts
- **Data management panel** — export, import, and export-for-teacher controls

## 5. The MCQ Diagnostic Missions

100 questions. 5 missions. Every question tied to the science of learning.

| Mission | Title | Subtitle | Topics Covered | Questions |
|---------|-------|----------|----------------|-----------|
| 1 | The Launchpad | Crisis & Brain Science | Neuroplasticity, memory consolidation, the forgetting curve (Ebbinghaus), focused vs. diffuse thinking modes, stress and learning, retrieval practice | 20 |
| 2 | Navigation Systems | Study Methods | Pomodoro Technique, Cornell Notes, Feynman Technique, Leitner system, interleaving, dual coding, spaced repetition | 20 |
| 3 | Galactic Engineers | Thinkers & Habits | Deep Work (Cal Newport), habit stacking (James Clear), cognitive load theory, identity-based habits, the Zeigarnik Effect, GTD system | 20 |
| 4 | Avoid the Black Holes | Systems & Tactics | Procrastination science, backward planning, focus zones, weekly review, ABCDE prioritisation, mock testing, batching, MoSCoW method | 20 |
| 5 | Life Support & Hero State | Wellbeing & Mastery | Sleep and memory consolidation, box breathing, anxiety reappraisal, nutrition for cognitive performance, growth mindset, the Yet Game, exercise and learning | 20 |

**Access:** All five missions are accessible at any time — there is no sequential lock. Missions can be retried. Each submission (including retries) increments the `diagnosticsCompleted` rank evidence counter.

**After each mission:**
- Individual answers are reviewed with correct/incorrect feedback
- Knowledge gaps (topics with incorrect answers) are saved and flagged
- A reflection prompt invites the student to write about what they plan to do with the result
- The student's best score per mission is retained for display

## 6. Teen-Focused Design

The following features were specifically designed with the realities of teenage study life in mind.

| Feature | What It Does | Why It Matters |
|---------|-------------|---------------|
| **Dark Mode Default** | App launches in dark mode; preference saved permanently | Reduces eye strain during evening study; matches the aesthetic teenagers already prefer |
| **Phone-Lock Nudge** | Notification fires when a Pomodoro session starts | The single biggest distraction during study is the phone on the desk. This is a direct behavioural prompt. |
| **Subject Tagging** | Each Pomodoro session is tagged with a subject | Surfaces subject-level patterns in the Stats view; helps students identify imbalances in their revision diet |
| **"What I Learned?" Modal** | After every session, a one-sentence prompt | Forces the student to articulate one concrete takeaway while the session is still fresh — a micro-retrieval practice moment |
| **Exam Countdown Widget** | Dashboard sidebar showing days remaining per exam; turns red at 7 days | Makes approaching deadlines visually inescapable |
| **Lofi Music Link** | One tap from the timer panel to a YouTube lofi playlist | Removes the excuse of "I need to find music before I start" |
| **Not Yet Button** | Third option on flashcards alongside Again and Got It | A student who does not know something should not feel penalised for admitting it. The Yet reframe makes the gap feel navigable, not shameful. |
| **Study Buddy System** | Generate a base64 buddy code (rank + stats); paste a friend's code for side-by-side comparison | Social comparison is a powerful motivator for teenagers. This channels it constructively. |
| **Shareable Progress Report** | A URL with encoded stats; readable by anyone (parent, teacher, coach) without an account | Parents can see evidence of real study. Students can be proud of it. |

## 7. The Teacher Dashboard

The Teacher Dashboard is a separate web page (`/teacher`) designed exclusively for educators. It requires no login, no account, and no per-student subscription.

### The Student-to-Teacher Workflow

1. **Students use Synapse as normal** — any device, any browser, fully offline if needed
2. **Student exports their data** — from Stats → Data Management → "Export for Teacher" button
3. **Student enters their name** when prompted and saves the downloaded `.json` file
4. **Teacher opens `/teacher`** and drag-and-drops all received files at once
5. **Dashboard populates instantly** with the full class overview

No data passes through any server. Everything happens in the teacher's browser.

### What the Dashboard Shows

**Class Summary (top panel)**
- Total students imported
- Average sessions per student
- Average total focus time
- Highest rank achieved in the class

**Rank Distribution Chart**
- Doughnut chart showing how many students are at Space Cadet, Second Officer, and Flight Commander

**Activity Comparison Chart**
- Bar chart comparing session counts across all students

**Students Needing Attention**
- Auto-flagged list of students with fewer than 5 sessions or zero blurts recorded
- Designed to surface struggling or disengaged students before they fall too far behind

**Student Roster**
- Full sortable table: name, rank, sessions, total focus time, flashcards advanced, blurts, active days
- Click any student to open their individual profile

**Individual Student Profile**
- Full rank evidence bars showing progress toward the next rank
- Activity breakdown by tool
- Timeline of recent sessions

### Printable Progress Reports

Each student profile includes a **Print Progress Report** button. This generates a clean, printer-friendly summary:
- Student name and current rank
- Rank evidence bars with current vs. required values for every criterion
- Activity stats (sessions, focus time, active days)
- Technique usage breakdown

These reports are designed for parent-teacher conferences, student check-ins, and school reporting requirements.

## 8. AI Study Planning — Mission Architect

The Mission Architect is the AI-powered module that generates a structured, personalised study plan. It is powered by Google Gemini (model: `gemini-2.0-flash`).

### How It Works
The student fills in a brief:
- Subject or topic
- Exam or deadline date
- Daily study time available
- Current level of confidence
- Any specific concerns or weak areas

The Mission Architect sends this to the Gemini API and returns a structured weekly plan with day-by-day tasks, technique recommendations, and priorities.

### API Key Setup (BYOK)

Synapse uses a Bring Your Own Key (BYOK) model. Each user sets up their own free Google API key:

1. Go to [Google AI Studio](https://aistudio.google.com/apikey)
2. Sign in with any Google account (free)
3. Click **"Create API Key"**
4. In Synapse, click the settings gear icon and paste the key into the API Key field
5. Click Save — the key is stored in the browser only and never leaves the device

If no user key is configured, Synapse falls back to a server-side proxy using the operator's key (set via environment variable `GEMINI_API_KEY`).

## 9. Data, Privacy and Storage

### Where Data Lives

All student data is stored in the browser's `localStorage`. No data is sent to any external server unless:
- The student explicitly configures Firebase (for cross-device sync)
- The student uses the Mission Architect (which sends only the study brief text to the Gemini API)

### localStorage Key Reference

| Key | Stores |
|-----|--------|
| `synapse_decks` | Flashcard deck metadata |
| `synapse_cards` | All card content and Leitner box positions |
| `synapse_pomodoro_log` | Pomodoro session history |
| `synapse_blurt` | Blurting method session records |
| `synapse_eisenhower` | Eisenhower Matrix tasks and reflections |
| `synapse_recal` | Dopamine Recalibrator session logs |
| `synapse_babel` | Babel Fish exercise records |
| `synapse_yet` | YET Growth Shield concept logs |
| `synapse_rank` | Rank level and all rank evidence counters |
| `synapse_mcq` | MCQ attempt history, scores, and gaps |
| `synapse_exams` | Exam countdown entries |
| `synapse_wil` | "What I Learned?" session log entries |
| `synapse_theme` | Theme preference (dark/light) |
| `synapse_gemini_key` | User's Gemini API key (BYOK; never exported) |
| `cornellNotes` | Cornell Notes panel HTML content (legacy key) |

### Export and Import

From the Stats page → Data Management:

- **Full JSON Export** — downloads all `localStorage` data as a single `.json` file (excluding the Gemini key for security)
- **Import** — restores all data from a previously exported file; overwrites current data
- **Export for Teacher** — same as Full Export but prompts for the student's name and embeds it in the file; designed for classroom data collection

### Privacy Notes

- The Gemini API key is excluded from all exports — it is never transmitted or shared
- No user account is required to use Synapse
- Firebase is entirely optional; without it, nothing leaves the browser
- The app functions fully offline with no network connection

## 10. Getting Started — Setup Guide

### Prerequisites
- Node.js installed (v16 or higher recommended)
- A modern browser (Chrome, Firefox, Edge, Safari)

### Step 1: Install Dependencies

```bash
npm install
```

### Step 2: Start the Server

```bash
node server.js
```

The server starts on port 5000 by default. Open your browser and go to `http://localhost:5000`.

### Step 3: (Optional) Configure the Server-Side Gemini Key

If you want to provide a fallback Gemini API key for users who have not set their own, set the environment variable:

```
GEMINI_API_KEY=your-api-key-here
```

On Replit, this is set in the Secrets panel. On other platforms, use your preferred environment variable method.

If this variable is not set, the Mission Architect will only work for users who have entered their own API key in Settings.

### Step 4: (Optional) Set Up Firebase Cloud Sync

Firebase enables cross-device sync for Cornell Notes and rank data. It is not required for any other features.

1. Go to [Firebase Console](https://console.firebase.google.com/) and create a project
2. In **Project Settings** → **Your apps**, click the web icon (`</>`)
3. Register an app name and copy the config values
4. Open `config.js` and replace the placeholder values:

```javascript
var SYNAPSE_FIREBASE_CONFIG = {
    apiKey:            "your-actual-api-key",
    authDomain:        "your-project.firebaseapp.com",
    projectId:         "your-project-id",
    storageBucket:     "your-project.firebasestorage.app",
    messagingSenderId: "123456789",
    appId:             "1:123456789:web:abc123"
};
```

5. In Firebase Console → **Authentication** → enable **Anonymous** sign-in
6. In Firebase Console → **Firestore Database** → create a database (start in test mode)
7. Restart the server and reload Synapse — Settings will confirm "Connected"

Without `config.js` configured, Synapse detects the placeholder values and runs silently in offline-only mode. No error is shown to the user.

### Step 5: Share with Students

Share your deployment URL with students. They visit the URL in any browser, and the app is immediately functional — no download, no account, no sign-up required.

## 11. Hosting Options

### Replit (Recommended for Getting Started)
The app is pre-configured for Replit deployment. Start the `node server.js` workflow and share the Replit URL. Replit's Deployments feature provides a persistent production URL.

### Run Locally (Development)
```bash
npm install
node server.js
```
Open `http://localhost:5000`

### Host on Any Node.js Server
Deploy `server.js` to any platform that supports Node.js — Railway, Render, Fly.io, DigitalOcean App Platform, AWS, etc. Ensure `GEMINI_API_KEY` is set as an environment variable if you want the server-side AI fallback.

### Host as Static Files (No Server)
Synapse is primarily a client-side application. The Express server is only needed for:
- Serving static files
- The server-side Gemini API proxy fallback

If you host the HTML files on Netlify, Vercel, or GitHub Pages, the app will work fully — but the server-side AI proxy will not be available. Students can still use the Mission Architect by entering their own API key in Settings.

### Open Directly in a Browser
For completely offline use, open `index.html` directly in any modern browser. All features work except the server-side AI proxy.

## 12. Customisation Guide

### Rebranding
- **App name:** Search for `Synapse` in `index.html` to replace all instances
- **Page title:** `<title>` tag near the top of `index.html`
- **Nav bar label:** The primary `<h1>` and logo text near the top of `<body>`

### Visual Appearance
- **Accent colour:** All interactive elements use Tailwind's `orange-600`. A global find-and-replace of `orange-600` (and `orange-500`, `orange-700`) with another Tailwind colour class will re-theme the app. Confirm `orange-50`, `orange-100`, `orange-200`, `orange-800`, `orange-900` background tints are also updated for full consistency.
- **Dark mode:** Dark mode is enabled by default. This is controlled in the Tailwind config block at the top of `index.html` (`darkMode: 'class'`) and by reading `synapse_theme` from `localStorage` on app init.

### Content
- **Motivational quotes:** Edit the `STUDY_QUOTES` array in the JavaScript section of `index.html`
- **Subject tags:** Edit the `<option>` elements in the subject dropdown (search for `timerSubject` in `index.html`)
- **AI model:** Change the `AI_MODEL` constant at the top of the `<script>` block to use a different Gemini model version

### Rank Thresholds
Rank requirements are defined in the `RANK_REQUIREMENTS` object in `index.html`. Thresholds can be adjusted, though any change will affect all existing users on that deployment.

## 13. Technical Architecture

### Overview

| Layer | Technology |
|-------|-----------|
| Frontend | Single-page application — all in `index.html` |
| Styling | Tailwind CSS via CDN (class-based dark mode) |
| Charts | Chart.js via CDN |
| Backend | Express.js v5 — static files + Gemini API proxy |
| Primary storage | Browser `localStorage` |
| Optional cloud | Firebase Firestore (anonymous auth) |
| AI | Google Gemini API (`gemini-2.0-flash`) |
| External data | Open Trivia Database (Quiz Arena only) |

### Design Decisions

**No build system.** All dependencies are loaded via CDN. There is no Webpack, Vite, or TypeScript. The entire frontend is a single HTML file. This makes the app trivially easy to host, modify, fork, and debug.

**Offline-first.** The app is fully functional with zero network connectivity. `localStorage` is the primary data store. Network features (Firebase sync, Gemini AI, quiz trivia) degrade gracefully when unavailable — the app does not error, it simply disables those features.

**Monolithic frontend.** `index.html` is approximately 2,800 lines. This is an intentional trade-off for simplicity of deployment and modification. There is no component system, no module bundler, and no framework overhead.

**BYOK AI model.** The Gemini API key model protects the operator from unexpected API costs while keeping the AI feature genuinely useful. Students who want full AI access set up a free key in under two minutes.

**Firebase as a progressive enhancement.** Firebase is not initialised unless `config.js` contains real credentials. The app detects placeholder values at startup and silently falls back to offline mode. No Firebase SDK calls are made in offline mode.

**Security.** All user-generated content rendered to the DOM passes through the `esc()` utility function, which escapes HTML entities to prevent XSS attacks. The Gemini API key is excluded from data exports.

### Server Routes

| Route | Method | Description |
|-------|--------|-------------|
| `/` | GET | Serves `index.html` |
| `/demo` | GET | Serves `demo.html` |
| `/teacher` | GET | Serves `teacher.html` |
| `/api/gemini` | POST | Proxies requests to the Gemini API using the server-side key |
| `/*` | GET | Static file fallback for all other assets |

All routes use `no-cache` headers to ensure students always receive the latest version.

### Environment Variables

| Variable | Required | Purpose |
|----------|----------|---------|
| `GEMINI_API_KEY` | Optional | Server-side Gemini API proxy. If not set, the proxy endpoint returns an error; BYOK still works. |
| `PORT` | Optional | Server port (default: 5000) |
